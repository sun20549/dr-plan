## 🔧 公式破解的修正

### 修正 1:備忘錄筆誤(`XWv^(n-1)` → `XWv^n`)
- 平準費率(level)Q 應該 = 1.0,但用備忘錄寫的 `XWv^(n-1)` 跑出 1.029
- 改成 `XWv^n_pay` 後,所有平準費率商品 Q = 1.0000 整齊 ✓
- **這是 Excel 公式列轉述時容易看錯的位置** — 已在 app.js 註解寫明

### 修正 2:D 序列邊界(填 n+1 個非零值)
- 一開始用「30 個 D」(D1..D30)算平準費率,Q 跑出 0.9718 仍非 1
- 推導發現:D 序列實際應填到「D[n+1]」(共 n+1 個非零值)才能讓 Q=1.0000
- 即 `buildDseq` 中 `fillCount = Math.min(n_pay + 1, 120)`

### 修正 3:首/續年雙費率(XCF 是這類商品)
- XCF 的 `XCFM/XCFF` 是「首年費率」,`XCFM2/XCFF2` 是「續年費率」
- D 序列要 D[0] 用首年表、D[1+] 用續年表
- 沒做這個修正時 XCF Q=10.6,做了後 Q=43.5(更接近備忘錄寫的 23.7 量級)
- XHC 也是這樣設計(本系統暫無 XHC)

## 📐 完整 XWA 公式(校正版)

### 單張險種

```
單張 XWA = ROUND( ROUND(G × P × Q / 100000, 2) × 主約繳別係數, 0 )
```

### 變數

| 變數 | 意義 | 取得方式 |
|---|---|---|
| **G** | 該險種年繳保費 | row.fee(已含保額,不含繳法折讓) |
| **P** | XWA 費率 | `WAIVER_PREM_DB[XWA{要保人年齡}{M\|F}][n_pay]` |
| **Q** | NPV 比值 | 由 D 序列計算(見下) |
| **主約繳別係數** | 年=1, 半年=0.52, 季=0.262, 月=0.088 | XWA_CONSTANTS.PAYMENT_FACTOR |
| **n_pay** | 該險種剩餘繳費年期 | 主約看「30年期」, 附約用 `maxAgeContinuous - startAge` |

### Q 公式(校正版)

```
XWv = 0.99009900990099  (= 1/1.01)

NPV = Σ D[i] / 1.01^i,  i = 1..119

Q = (0.5 × D₁ × XWv^0.5 + NPV) /
    (0.5 × D₁ × XWv^0.5 + D₁ × (1 - XWv^n) / (0.01 × XWv) × XWv)
```

### D 序列構造規則

| 商品類型 | D 序列 |
|---|---|
| **平準費率(level)** | D[0..n] 全填 D₁,之後填 0 |
| **自然費率(natural)** | D[0] = ratesMap[startAge], D[i] = ratesMap[startAge + i] |
| **首/續年雙費率** | D[0] = firstYearRates[startAge], D[i] = renewalRates[startAge + i] |

填到 **n+1 個非零值**(才能讓平準費率 Q 嚴格 = 1.0000)
