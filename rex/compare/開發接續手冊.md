## 📦 專案目標

台灣保經業務員自用的儲蓄險商品比較工具。流程三步驟：
1. 輸入被保險人資料（姓名、性別、生日、年期、預算）
2. 看商品列表 + 預算對應保額
3. 比較解約金 / 身故金的逐年數據

從各保險公司的 Excel 試算表反向工程出精算引擎，建成本地資料庫。

## 🧠 三個精算引擎邏輯

### 1. taishin_v1（新光分紅型 — 完整精算）

```
保費 = 保額/1000 × GP × (1-折扣 0.01)
基本解約金 = 保額/1000 × CSV (Result_UV 表)
基本身故金 = MAX(保額×Corridor 係數, 保價金)
紅利累積：DIV.AD 累積成增清面額後計算
```
- 32 歲男 + USD 32,000 自動切金標準（Excel 原始數據）
- 適用：UPD061、UPD101

### 2. prudential_v1（保誠 — RV 表預含全紅利）

```
兩階段折扣：round(保額/10000 × GP × 0.99) × 0.995 後再 round
RV 表已含 DB/CV/PV/ARB/TB/ARB_PV/TBS_CV 全部欄位
解約總和 = CV + ARB_PV + TBS_CV
身故總和 = DB + ARB + TB
```
- 任何年齡都精準（不需自己累積紅利）
- 適用：ARLPLU71

### 3. twlife_v1（台灣人壽 — 單一基準逆推）

```
從「資料檢核頁」抽出 70 個保單年度逐年數據（cum_prem, cv_basic, cv_total, death_benefit 等 18 欄）
ratio = 用戶預算 / 基準保費
各年度數字 × ratio 縮放
```
- `is_exact = (sex===base_sex && age===base_age)`，否則 `is_estimated = true`
- UI 標 📐 估算（單一基準逆推，不同年齡是估算值）
- 適用：NUIW5203、NUIW6602

## 🐛 重要 Bug 修復史（避免新對話走回頭路）

### Sticky 表頭錯位 bug
- **症狀**：第三頁比較表的左側 meta 標題（商品名稱、代號…）跟右側資料對不上
- **失敗解法**：v4.0~v4.6 用寫死 top 值（不對）；v4.7 用動態量測（字體載入時序問題）
- **正解**（v4.10）：thead 9 列全進 thead，用 `layoutStickyHead()` 動態量測每個 row 高度後寫回 inline `top`，配合 `document.fonts.ready` + `setTimeout(100/400ms)` 雙重保險

### 雙層捲動 bug
- **症狀**：v4.10 的比較表外有個容器，要在容器內捲動，很卡（v4.12 截圖回報）
- **修法**（v4.12）：容器 `overflow-y: visible`，移除 `max-height: 78vh`，改用瀏覽器主視窗捲動。Sticky 自動黏到 viewport top。

### Console Messages 干擾
- **症狀**：預覽環境顯示「Console Messages」黑色區塊（v4.12 截圖回報）
- **修法**（v4.13）：移除資料庫載入時的兩行 `console.log`

## ⚙️ 整合新商品的標準 SOP

如果是「公式驅動 + 業務員已填參數」的 Excel：

1. 用 LibreOffice 重算：
   ```bash
   cd /home/claude/products
   TMPHOME=/tmp/lo_$$
   mkdir -p $TMPHOME
   soffice --headless --norestore --nologo \
     -env:UserInstallation=file://$TMPHOME \
     --convert-to xlsx --outdir recalc \
     <輸入檔.xlsx>
   ```
2. 用 openpyxl 讀「資料檢核頁」或「明細版」抽資料
3. 寫進 `<company>_db.json`
4. 整合進 `v4_13.html` 的 `window.INSURANCE_DBS`
5. `PRODUCTS` 陣列加新商品條目（含 `engine`、`unit_size`、`min_sa` 等）
6. 若是新引擎類型，在 dispatcher 裡加新分支
7. Node.js 端到端驗證
8. 輸出新版本 `v4_X.html`
