## ✅ 這輪做的(v23 → v25 整合)

### Bug 修復:v2_full key 格式錯誤(影響 12 個檔)
v23 寫的 extractor 把 plan code 也包進 gp_table/pv_table key,變成 9~14 字元(如 `06ULI937M`、`01TLD3/TLD516M`),JS 引擎用 5 字元 key 查找(如 `0637M`)→ 找不到 → 退回 v1 → 只顯示 20 年。

修復:寫 fixer 用正則把 plan code 從 key 中移除,12 個檔的 key 全部統一成 `<period(2)><age(2)><sex(1)>` 5 字元。

修正的商品(用戶反映「21 年起空白」就是這個):
- 台灣人壽 NUIW4703 / NUIW5203 / NUIW5903(吉美富 / 超美利 / 吉美得)
- NUPW0102 美紅旺_v5 / NUPW0401 美紅勝利 / NUPW0701 美紅富利
- NUIW6502 保利美
- 金得利

### JS 引擎升級:支援 gp-only v2_full 模式
原引擎要求 gp_table + pv_table 都有,缺一就退 v1。
新引擎:
- gp_table 有 → 保費精準計算(對任何年齡/性別)
- pv_table 缺 → 從 schedule_at_base 反推 pv 序列
- 結果:保費 100% 對齊 Excel,CV/PV 是估算

這讓富邦這類「Excel 只有 GP 沒有 PV/CV table」的商品也能跑 v2_full(部分精算)。

### 富邦 PDATA GP 抽取
- 寫 fubon GP extractor(支援中文/英文 header)
- 成功抽出 FAT 174 個費率
- FAK1 150 個費率(已是 v2_full)
- FBM/FBO 抽取邏輯有 bug 待 debug
