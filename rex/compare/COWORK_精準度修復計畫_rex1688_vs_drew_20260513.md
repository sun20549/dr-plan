## 六、4 個系統性問題(架構層面要修)

### S-1 ⚠️ 命名規範不統一
- 有的 JSON 用 KEY(`UPD061.json`)
- 有的用中文名(`保利美.json`、`吉美富.json`、`美紅旺_v5.json`)
- 有的少前綴(`RPISWLB.json` 應為 `6RPISWLB.json`)
- 有的帶版本後綴(`美紅旺_v5.json`)

**修法**:
1. 統一規定 — 檔名 = KEY = manifest.key = 商品建議書代號
2. 寫一支 `rename_to_key.py` 把所有中文名 JSON 改名,並更新 manifest

### S-2 ⚠️ UPD061 載入失敗(alert 跳過)
- rex1688 跑試算時跳出原生 `alert("以下 1 個商品載入失敗,將從比較中跳過: UPD061 ...")`
- 推測:`UPD061.json` 在頂層也在 `skl/`,manifest path 指錯

**修法**:檢查 manifest 中 UPD061 的 path 欄位,確認 fetch 200。前端把 alert 改成右下角 toast,別阻塞操作

### S-3 ⚠️ 不該 alert 阻塞 UI
- 任何商品載入失敗,UI 用 toast 或 inline 標籤顯示,別跳原生 alert

### S-4 ⚠️ 公司分類錯誤(skl vs tsl)
- 本機 `data/skl/UPD061.json` — UPD061 在 Drew 屬於 **台新人壽**(「台新」),不是新光(「新光」)
- 雖然兩家現已合併為「台新新光金控」,但商品代號歸屬不同
- 建議子目錄改成 `tsl/`(Taishin Life)放台新商品,`skl/`(Shin Kong Life)放新光商品
