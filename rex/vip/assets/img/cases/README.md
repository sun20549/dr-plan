# 案例對帳單截圖目錄

## 檔名規則

把對帳單截圖丟進這個資料夾即可，網站會**自動偵測**並顯示。

**命名格式**（**小寫**案例編號 + 序號）：
- `u0033-1.jpg` — U0033 的第 1 張對帳單
- `u0033-2.jpg` — U0033 的第 2 張
- `u0033-3.png` — U0033 的第 3 張（也支援 png）
- `k3007-1.jpg` — K3007 的第 1 張
- `f2040-1.webp` — F2040 的第 1 張（也支援 webp）

## 支援格式

- `.jpg` / `.jpeg`
- `.png`
- `.webp`

## 自動偵測機制

每個案例頁載入時，會依序試探：
```
u0033-1.jpg → u0033-1.png → u0033-1.webp
u0033-2.jpg → u0033-2.png → u0033-2.webp
...直到第 8 張
```
找到的圖就會顯示在「真實對帳單佐證」區塊（含 lightbox 點擊放大）。

## 建議

- 圖檔大小：盡量壓到 200KB 以下（用 [TinyPNG](https://tinypng.com) 或 [Squoosh](https://squoosh.app)）
- 寬度建議 ≥ 1200px，網頁會自動縮放
- 個資/身分證號等敏感資訊請先打碼

## 範例

如果你 U0033 有 3 張對帳單，K3007 有 2 張，這個資料夾就放：
```
u0033-1.jpg
u0033-2.jpg
u0033-3.jpg
k3007-1.jpg
k3007-2.jpg
```

不需要修改任何 HTML 程式碼，重新整理頁面就會自動出現。
