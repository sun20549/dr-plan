# Workflow · 商品上架驗證

## 鐵則

**對照官方 PDF 至少 3 案例,誤差 ≤ 0.01 USD 才能上線。**

使用者非常堅持這點。曾發生美鴻世代 F35 yr 1 算 112,324 對不上 PDF 202,184,
後來發現是少了 corridor_criteria 公式。

## 3 案例範本

| 類型 | 為什麼要驗 |
|------|------------|
| **未滿 15 歲**(如 M5 150 萬) | 驗 funeral_cap 上限 + age 16 那年的解除 |
| **成人低面額**(如 F35 30 萬) | 驗年齡別 criteria 公式 + 折扣 |
| **成人高面額**(如 M55 100 萬) | 驗高保費折扣級距 + TDD/TDS 偏移 |

## 對照欄位

至少:`A 當年保費`、`B 累計保費`、`C 身故`、`D 解約`、`H 終期身故`、`I 終期解約`、首期折扣後保費。

抽樣年度:`yr 1, 2, 3, 5, 7, 10, 15`(找轉折點)。
